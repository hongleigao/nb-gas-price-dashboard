# **NB 省油价预测与看板系统详细设计文档 (v7.0)**

## **1\. 系统概述 (Overview)**

本项目旨在建立一个 NB 省汽油价格的预测与历史追溯面板。系统基于 Serverless 架构，遵循“低成本、免运维、高可用”的原则，核心逻辑严格贴合 NB EUB 的定价规则。

**技术栈：**

* **数据源**：EUB 官网 (Excel) \+ Yahoo Finance (yfinance API)  
* **任务调度 & 计算**：GitHub Actions \+ Python 3  
* **数据库**：Cloudflare D1 (SQLite)  
* **后端 API**：Cloudflare Workers (Node.js)  
* **前端展示**：React \+ Vite \+ ECharts \+ Tailwind CSS (静态托管于 Cloudflare Pages)

## **3\. 核心业务规则 (Business Rules)**

* **定价周期**：常规周期为上周四至本周三（5个交易日），每周四晚发布，周五凌晨 00:01 生效。  
* **熔断机制 (Interrupter)**：单日极端波动 ![][image1] 6.0 ¢ 或多日累计偏离 ![][image1] 5.0 ¢ 时触发。  
* **法定静默期**：星期二 (Tue) 与星期三 (Wed) 绝对不触发熔断。  
* **市区预估价计算**：Pump Estimated Price \= Max Retail Price \- 5.5 ¢  
* **基准对齐 (Benchmark)**：计算本周变动时，必须以**上一常规周期**的 5 日 RBOB\_CAD 均值为基准。  
* **最终预测值公式**：常规 5 日均线预测变化值 \- 周期内已发生的熔断变化值 (Interrupter Variance)

## **4\. 数据库安全与性能约束 (Database Schema & Indexes)**

**硬性约束**：所有 D1 查询**必须**使用参数化查询（Parameterized Queries），严禁使用字符串拼接 SQL。

### **表 1: market\_data**

CREATE TABLE market\_data (  
    date TEXT PRIMARY KEY,  
    rbob\_usd\_close REAL NOT NULL,  
    cad\_usd\_rate REAL NOT NULL,  
    rbob\_cad\_base REAL GENERATED ALWAYS AS (rbob\_usd\_close \* cad\_usd\_rate) VIRTUAL,  
    is\_weekend INTEGER DEFAULT 0   
);  
CREATE INDEX idx\_market\_date ON market\_data(date);

### **表 2: eub\_prices**

CREATE TABLE eub\_prices (  
    effective\_date TEXT PRIMARY KEY,  
    published\_date TEXT NOT NULL,      
    max\_price REAL NOT NULL,           
    is\_interrupter INTEGER NOT NULL DEFAULT 0,  
    interrupter\_variance REAL DEFAULT 0,  
    created\_at DATETIME DEFAULT CURRENT\_TIMESTAMP  
);  
CREATE INDEX idx\_eub\_effective\_desc ON eub\_prices(effective\_date DESC);

## **5\. 后端 API 契约与本地开发约束 (API & DevEnv)**

### **5.1 协议与 CORS 约束**

* 生产环境的 CORS Access-Control-Allow-Origin 必须限制为 Pages 域名。  
* 生产响应头强制注入：Cache-Control: public, max-age=1800。

### **5.2 Endpoint: GET /api/v1/cycle/current**

Worker 必须动态计算 benchmark\_price (上一周期的 RBOB\_CAD 均值)。

### **5.3 Endpoint: GET /api/v1/history?days=90**

API 并行查询两张表，吐出指定天数的双轨数据，不作插值计算。

## **6\. 前端业务逻辑与极简架构约束 (Frontend Core Logic)**

**硬性约束**：严禁引入 Redux 等重型状态管理库。

### **6.1 时区强制锁定与偏差计算**

使用 Intl.DateTimeFormat 锁定 America/Moncton 时区。计算：

* 单日偏离值 (daily\_variance) \= 当天绝对价 \- 昨天绝对价  
* 累计偏离值 (cumulative\_variance) \= 当天绝对价 \- benchmark\_price

### **6.2 熔断风险评估指示器 (Interrupter Risk Assessment)**

前端实时渲染风险状态标签（无需后端干预）：

* **Triggered/Alert**：abs(daily\_variance) \>= 6.0 或 abs(cumulative\_variance) \>= 5.0。  
* **High**：未触发，且 abs(cumulative\_variance) \>= 4.0。  
* **Medium**：未触发，且 3.0 \<= abs(cumulative\_variance) \< 4.0。  
* **Low**：abs(cumulative\_variance) \< 3.0。

## **7\. UI/UX 裁剪规范与响应式约束 (UI/UX Specifications)**

为契合 Serverless 架构，严格禁止前端开发虚假元素：

* **禁止**开发系统监控/设置流心跳页。  
* **禁止**开发预测置信度、AI 市场洞察、交易量(Volume)、以及 Freight LOGISTICS 物流系数等衍生指标。  
* **禁止**使用平滑曲线代替 EUB 的官方阶梯图线 (Step Line)。

## **8\. 页面视图与组件物理文件映射表 (View to Component Mapping)**

前端开发人员必须严格按照以下对应关系将 UI 设计稿落地为 React 组件，确保路由状态与业务逻辑解耦：

| UI 设计稿视图 | 物理组件路径 | 路由与交互约束 | 核心功能点 |
| :---- | :---- | :---- | :---- |
| **预测主页 (Market)** | web/src/components/HeroBoard.jsx | 一级页面。底部展示导航栏，MARKET 高亮。 | 请求 /cycle/current API，展示当前限价、预测差值。计算风险文案。 |

## **9\. 数据流异常熔断约束 (Python Script Constraints)**

在 scripts/update\_daily.py 中：

1. **yfinance 幽灵数据**：通过 pandas.isna() 过滤假数据。  
2. **汇率缺失**：若缺少 CAD 汇率，脚本必须执行前向填充 (Forward-fill)。
